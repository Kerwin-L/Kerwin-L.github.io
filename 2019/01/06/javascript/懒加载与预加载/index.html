    懒加载与预加载

    一、懒加载

    1.什么是懒加载?为何要用懒加载

    懒加载就是延迟加载，

    可将其设置为图片在可视区域范围的时候才真正开始加载图片，从而提高页面的加载和响应速度。

    2. 懒加载的原理

    img 标签如果没有 src 属性，浏览器就不会发送图片去下载图片。

    可以通过在初始时候设置图片的 加载gif ，然后在其其他属性（ data-original） 中添加真实img 图片URL，然后在监听滚动事件，通过判断是否进入可视范围，若进入则将其 data-original 的值赋予 src ，实现延迟加载。

    3. 懒加载实现

    通过 node.getBoundingClientRect() 获取元素的 上下左右 分别相对于浏览器视窗的距离，通过判断

    PS. HTMLElement.dataset

    属性允许在 读入或者写入中访问 dom 元素的 自定义属性(data- )

    即： 可以通过 document.getElementById(‘data’).dataset.original 获取 ‘xxx.jpg’ 字符串

    二、预加载

    1. 什么是预加载

    将所需要的资源提交加载到本地，这样后面需要用到即可直接从缓存中获取资源
