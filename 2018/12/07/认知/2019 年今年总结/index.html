速读笔记

Posted on 2018-12-07 Edited on 2022-01-30 In Learning_Method

2019 年总结

今年是这辈子最幸运的一年

1. 毕业前实习的经历

对于当时我的状态，真正意义上的进行前端的学习应该是 2018 年 8 月份，从李老师那边开始正式实习，历时 6 个月。

其实这段经历并没有那么不堪，其实也算是我快速获取概念的时候，也借此机会学到了 docker / nginx / vue / vue-cli / vue-router / mongodb / node-express等东西,整个半年也在三个项目（数院财务系统 / 文康的系统 / universe 服装商城）中不断的进行切换。

也在这个阶段亏损了自己的第一个投资 universe, 作为这个项目的所谓的技术负责人，明白最多的是，做技术不仅只是你干技术就可以了，技术只是整个环节中的一环罢了，更重要的是团队氛围 / 团队成员的成长 / 成功带来的反馈，当团队足够小的时候，各方面的能力更为重要。也正是因为这个项目让我从对方的角度出发去思考问题，从产品本身的角度去思考产品的技术设计，需要全面的观察整个产品

2018 年 9 月份的时候才开始接触到 vue 这种框架，而在这之前都是思路一直纠结在哪种样式库更好一点。虽然一直是在担心这个那个，但是一直都停滞不前，其实现在也是这样！！这个该如何解决呢？。

在完成项目后，总结整个项目的开发模式！！！，我认为这个是解决心中一直“有疑惑”的开发模式（就是每次开发设计的时候，会向着人是否这样设计是好的）。

2. 加入商汤，在海翔进行实习！

这算是我真正意义上认识到了大公司，即使进来时候仅有我和另外一位小伙伴，不过晗姐让我明白了什么才是真正的领导。有十分明确的目的以及计划，不俗的控场能力。这在更多是接触更多是项目上的东西。（也在这个时候接触了 react + ts + antd + dva 这些概念），真的，你的身边真的不缺资源！！！！，焕州 / 忠键 / 冠亮都是比你经验丰富的人，其实你缺的只是一个坦诚开放，无所不惧，虚心求学的心而已）

3. 正式加入商汤！！加入辉哥底下！！

这是我人生的关键转折点！！！也正是这个机会我有机会去跻身如互联网行业，不然以当时的那个那个实力，可能真的到了下层的传统的行业了。这时候真的要感激辉哥以及朝滨，不是他们赏识我的整体的能力的话，凭借我的技术硬实力的话可能还是进不了商汤的。（其实到现在我还是些许觉得我的技术匹配不上我的身价，还好其他方面的能力 ok 的，不过自身的实力还是最重要），记得以后你能帮到辉哥/朝滨的事一定要义不容辞

4. 入职到现在

入职到现在历时 7 个月，到现在为止最可惜是项目还没有正式的上线。在这段时间里经历了很多，也认识了很多不一样的大佬

  1. 入职团队拓展 8 月份

在这时候认识到了身边的很多优秀的人，有博士，基本都是硕士级别的人，但其实我欠缺是不会去表达自己的想法(还是我自己本身就是没那么多的想法？其实在不懂的时候就问是对自己最快的提高！你可以从别人的角度更全面的去掌握整个概念)。而在这个阶段其实有很多可以去逼自己一把的机会但是却没有，这点比较可惜，其实更多是自己的自洽能力？

  1. 黑客马拉松 10 月份

最大的好处是到处去玩了 😂，终于去到了北方啦！！终于见识到外面的世界啦！在这个阶段我算是完成了我该完成的任务，得到了个三等奖，虽然整个项目并没有太多的技术含量，但仅有我一名工程师的我来说，整体还是值得鼓励的。也在这时候跟露露、逸如、lucy 深度认识了下，也发现逸如这人跟我很相似，当时也非常能聊，这个晚点说。

  1. 上海年会 12 月份

这个时候的两个表演也让我见识到了很多之前没接触过的东西，终于踏上了舞台了！！！还在上海走了那么多的路。第一次到了录音房，各种幕后，以及团队人的各种支持。虽然有着不完美的结局，这也正式为我下次的表演做铺垫吧 😄。

以上这些都是算是专业以外的软能力和见识的提高吧。其实中间享受的是新体验带给我那种快感，第一次站在舞台上的那种感觉，聚光打在你身上的时候那一瞬间，感觉路途带来的疲惫都是美妙的。这算是我不多向往的东西。所以这段时间的练习就很有必要了！！

在技术层面，从开始到现在的 7 个月，其实大大小小的书都是看到了一半都结束了，其实这个阶段还是自己之前的想法： 现在学习的东西都是为了后面的 5G 打下基础，其实可以发 ？？ 表情包了。这种目标对与你之前看过的书来说，这个目标根本不算合格，首先需要对目标不够具体，其次目标也不够量化，导致这样的目标实则为一个虚拟的概念而已。

今年我想达到的目标是：

  1. 能够很轻而易举的带领团队 10 人以上的团队。
  2. 能够在平台上面拿下 1000 名粉丝
    1. 给自己整理项目中的技术难点
    2. 锻炼自己的文字表达能力
    3. 总结自己的技术
  3. 总体的薪资能翻一翻

Key Result

而到现在为止，对比于入职前的我来说，技术上并没有什么突破型的开窍，还是在摸爬滚打着走。真的明白如果是真的有较于上阶段前端的成长的话，就是对一些新的概念的了解，dva / rxjs,其实我在开发的时候会明显感觉到几个问题的重要性

  1. 不同人之间的代码风格规范很重要（特别是在工具部分的时候，只要思路不清楚的话，整个对接起来就非常困难）
    1. 逻辑格式
    2. 编码规范
  2. 版本管理非常重要 （需要去看看好代码到底是什么）
  3. typescript 的使用真的非常重要（特别范型的概念，自己看起来就非常的不清晰）

现在还有一个关键，就是找到一个自己的技术方面真心佩服到五体投地的人！！
